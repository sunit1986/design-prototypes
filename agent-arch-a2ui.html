Translation Flow
1
Input Reception
A2UI receives agent output in one of three forms: structured JSON from tool calls, a streaming SSE token feed, or a complete NL response string. All three are normalized to an internal event stream.
2
Intent & Schema Parse
Intent Classifier determines the display category (data table, action card, narrative prose, etc). Schema Extractor then walks the payload to build a typed field map. Entity Mapper annotates domain-specific values.
3
UI Spec Generation
Layout Planner chooses the right spatial structure. Component Selector queries the design system registry for the best-fit component. Prop Builder generates a fully typed props object ready for rendering.
4
Design Token Application
Token Mapper injects the active design token set. Responsive Adapter adds breakpoint rules. Theme Resolver applies brand overrides and resolves the current color scheme.
5
Render & Emit
Virtual DOM Builder constructs a diffable tree for efficient updates. Animation Engine adds entrance transitions and streaming skeleton loaders. A11y Layer enforces ARIA compliance. Final output is emitted as a live component, exportable code, and a preview stream.
Key Components
🌊
Token Stream
Input · Ingestion
Consumes SSE events from the LLM provider. Buffers partial tokens and emits complete semantic units downstream so the render pipeline never blocks on an incomplete word.
SSEBufferingBackpressure
🔎
Intent Classifier
Parse · Core
Lightweight classifier (fine-tuned on display intent labels) that routes each payload to the correct rendering pathway in under 10 ms, enabling near-instant UI decisions.
ClassificationLow-latencyRouting
🧩
Component Selector
Spec · Matching
Queries a component registry using semantic + constraint matching. Scores candidates on data shape fit, interaction requirements, and accessibility compliance before selecting the winner.
RegistrySemantic MatchScoring
🎨
Token Mapper
Design · Styling
Maps abstract design tokens (color-primary, space-4, etc.) to resolved CSS custom properties at build time. Supports runtime token swaps for multi-brand and theme switching.
Design TokensCSS Custom PropsMulti-brand
🌳
Virtual DOM Builder
Render · Core
Constructs an immutable VDOM tree from the UI spec. Diffs against the previous tree on each streaming update to minimize DOM mutations and eliminate visual flicker during live rendering.
VDOMDiffingImmutable
♿
A11y Layer
Render · Compliance
Automatically infers and injects ARIA roles, live-region attributes, and focus management rules. Runs a WCAG 2.2 AA contrast check on every generated color pairing before output.
ARIAWCAG AAFocus Management
