How it works

Show the thumbnail first. On tap, swap it for a YouTube embed iframe with autoplay=1. Works because MCP App grants allow="autoplay; encrypted-media".

// 1. Thumbnail first (instant load)
<img src="img.youtube.com/vi/{ID}/maxresdefault.jpg"/>

// 2. On press — swap to live player
function playInline() {
  wrap.innerHTML = `<iframe
    src="youtube.com/embed/{ID}?autoplay=1"
    allow="autoplay; encrypted-media"
    allowfullscreen />`;
}

Why it fails

YouTube sets X-Frame-Options: SAMEORIGIN on all responses. MCP UI's sandbox cannot override this — the browser silently blocks the iframe.

// ❌ Always blocked — never use this
<iframe
  src="youtube.com/embed/{ID}?autoplay=1"
  allowfullscreen
></iframe>

// Browser error response headers:
X-Frame-Options: SAMEORIGIN
Content-Security-Policy: frame-ancestors 'none'

Why does this happen?

🔒 YouTube's Policy

YouTube sends X-Frame-Options: SAMEORIGIN on every page. Browsers refuse to render it inside a foreign-origin iframe — no exceptions.

📦 The MCP Sandbox

Both MCP App and MCP UI render inside a <iframe sandbox>. Nesting a YouTube iframe inside that triggers the same-origin block.

✅ The Right Pattern

Show a thumbnail first using img.youtube.com/vi/{id}/maxresdefault.jpg. On tap, swap to the embed. Works in every MCP context.
