Troubleshooting

uvx: command not found
Close and reopen Terminal after installing uv â€” it updates your shell's PATH. If Claude Desktop still can't find it, run the symlink fix below. 
sudo ln -sf $(which uv) /usr/local/bin/uv && sudo ln -sf $(which uvx) /usr/local/bin/uvx

Enter your Mac password when prompted â€” normal. Then restart Claude Desktop (Cmd+Q).

Tools not showing up in Claude or Cursor
Fully quit the app (Cmd+Q on Mac) â€” just closing the window isn't enough. Reopen after a complete quit. The tools appear in the MCP panel after a clean restart.
Server disconnects or "No such file or directory"
Claude Desktop launches in a restricted environment where your shell PATH isn't loaded. uv/uvx are installed in your user directory but not in /usr/local/bin/. The symlink fix above solves this permanently.
Slow first launch
Expected â€” uvx downloads the package and a minimal Python runtime the first time (~10 seconds). Everything is cached locally after that. Subsequent starts are instant.
