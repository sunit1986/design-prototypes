JDS Knowledge Server

One command handles everything — uv, git, config.
Just paste, restart, and go.

Recommended

Automatic Setup

Open Terminal and paste this:
curl -LsSf https://raw.githubusercontent.com/sunit1986/JioBharatIQ_Server/main/install.sh | bash

Installs uv, git (Xcode CLT on Mac), configures Claude Desktop & Cursor, and caches the server package. It will ask for your Mac password once (to make uvx visible to GUI apps).

After it finishes, restart Claude Desktop / Cursor (Cmd+Q and reopen).
Alternative — Manual Setup

Claude Desktop

Settings > Developer > Edit Config — paste this:
{
  "mcpServers": {
    "JioBharatIQ": {
      "command": "uvx",
      "args": ["--from", "git+https://github.com/sunit1986/JioBharatIQ_Server.git", "jiobharatiq-server"]
    }
  }
}

Requires uv already installed. Quit (Cmd+Q) and reopen Claude Desktop.

Cursor

Settings > MCP > Add Server — paste this:
{
  "mcpServers": {
    "JioBharatIQ": {
      "command": "uvx",
      "args": ["--from", "git+https://github.com/sunit1986/JioBharatIQ_Server.git", "jiobharatiq-server"]
    }
  }
}

Restart Cursor.

Claude Code (CLI)

Paste in terminal:
claude mcp add JioBharatIQ -- uvx --from "git+https://github.com/sunit1986/JioBharatIQ_Server.git" jiobharatiq-server
Test it — ask Claude: "What JDS button variants are available?"

How it works

Auto-updates Knowledge base fetches latest from GitHub on every startup. No action needed.
Assets Fonts, icons, animations served from GitHub CDN. No local files.
Requirements uv only (~30 MB). Bundles its own Python runtime.
First launch ~10 sec to download package. Cached after that — instant starts.

Force update

If you need the latest server code immediately:
uvx --reinstall --from "git+https://github.com/sunit1986/JioBharatIQ_Server.git" jiobharatiq-server

Knowledge base updates automatically. This command is only needed for server code changes.

Troubleshooting

uvx: command not found?
Close and reopen Terminal after installing uv. Then try the config again.
Server disconnects or “No such file or directory”?
Claude Desktop can’t find uv/uvx in its PATH. Run this once in Terminal to fix it: 
sudo ln -sf $(which uv) /usr/local/bin/uv && sudo ln -sf $(which uvx) /usr/local/bin/uvx

It will ask for your Mac password — that’s normal. Then restart Claude Desktop (Cmd+Q and reopen).

Tools not showing up?
Fully quit the app (Cmd+Q on Mac) and reopen. Just closing the window isn't enough.
